Challenge: Path Integration

Path integration is a mechanism that nesting animals can use to find their way back. It consists of maintaining a vector (i.e. the angle and distance) back to where they started. This challenge provides you with a path integration class, but you can write your own.

Sensors

Key Type Description
prevVelocityAchieved Vec2 The x/y translational change WRT the agent's center that was achieved in the last step. Note that the agent can request any speed, turn, or sidle it wants, but what actually is achieved is subject to things like physiological constraints, fatigue, illness, obstacles, or terrain. Also, the values have been rounded to 2 decimal places, since no sensor values should ever be considered exact.
prevAngularVelocityAchieved number The actual amount of orientation change made in the previous step. The value has been rounded to 2 decimal places, since no sensor values should ever be considered exact.
contact boolean Whether a contact was made with another object.

Actuators

Key Description Range
speed How fast the agent moves forward or backward. -0.5 ... 2
turn Rate of turning. -π/12 ... π/12
sidle How fast the agent moves side to side. -0.4 ... 0.4

Goal

The goal is to set out from the center, make contact with another object, and then return to the start. Other objects will appear at a given range of distance from the starting point at random angles.
