Challenge: Gather

This challenge provides you with two internal path integrators that give you distance and angle information to the target and the goal.

Sensors

Key Type Description
prevVelocityAchieved Vec2 The x/y translational change WRT the agent's center that was achieved in the last step. Note that the agent can request any speed, turn, or sidle it wants, but what actually is achieved is subject to things like physiological constraints, fatigue, illness, obstacles, or terrain. Also, the values have been rounded to 2 decimal places, since no sensor values should ever be considered exact.
prevAngularVelocityAchieved number The actual amount of orientation change made in the previous step. The value has been rounded to 2 decimal places, since no sensor values should ever be considered exact.
contactAngle number | undefined The angle WRT to the nose that contact was made between the agent and another body.
targetDistance number The distance in units from the center of the agent to the center of the target body.
targetAngleDiff number The difference between the agent's heading and target, or put another way, the amount the agent's heading should be changed to point toward the target position.
goalDistance number The distance in units from the center of the agent to the center of the goal.
goalAngleDiff number The difference between the agent's heading and goal, or put another way, the amount the agent's heading should be changed to point toward the goal.

Actuators

Key Description Range
speed How fast the agent moves forward or backward. -0.5 ... 2
turn Rate of turning. -π/12 ... π/12
sidle How fast the agent moves side to side. -0.4 ... 0.4

Goal

The goal is to push the target body back to the goal area.
